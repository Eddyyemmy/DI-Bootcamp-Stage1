Flexbox

What, exactly, is flexbox? Letâ€™s turn to the MDN documentation for an overview:

The CSS3 Flexible Box, or flexbox, is a layout mode providing for the arrangement of elements on a page such that the elements behave predictably when the page layout must accommodate different screen sizes and different display devices. For many applications, the flexible box model provides an improvement over the block model in that it does not use floats, nor do the flex containerâ€™s margins collapse with the margins of its contents.

Many designers will find the flexbox model easier to use.

  • Child elements in a flexbox can be laid out in any direction and can have flexible dimensions to adapt to the display space.
  • Positioning child elements is thus much easier, and complex layouts can be achieved more simply and with cleaner code, as the display order of the elements is independent of their order in the source code.
  • This independence intentionally affects only the visual rendering, leaving speech order and navigation based on the source order.

In conclusion,
Flexbox stands for flexible box. Itâ€™s a layout module for CSS aimed at giving you an efficient way to arrange, organize, and size website elements to create highly adaptive designs.

You can read more about flexbox in the MDN Documentation , in the LearnLayout website and in the scotch.io tutorial.


When to Use Flexbox

Flexbox is most appropriate for small-scale layouts and applications, such as:

  • navigation menus
  • card layouts
  • media items
  • web forms


Flexbox axis

The way flexbox works is quite simple on the surface: You have a container (flex container) with children (any elements contained within, called flex items), which are placed along flex lines.

Lines and items can be manipulated in layout, size, spacing, and more along both the vertical and horizontal axis using a multitude of operators. This allows you to best take advantage of the available space and lets elements arrange themselves automatically according to it.


  • main axis â€” This is the axis at which the items are laid out. Important: it can be both vertical or horizontal, depending on the flex-direction property.

  • cross axis â€” The axis perpendicular to the main axis. Its direction, too, depends on how the main axis is defined.

  • writing-mode â€” Allows you to switch the direction of writing from left-to-right to right-to-left or even to vertical. Itâ€™s a work in progress with little to no browser support, however, itâ€™s important to know for some of the properties further below.


Flexbox properties


I. Flex properties on the flex container

Display

Weâ€™ve seen this property before. It turns out that this property is of fundamental importance when you want to use flexbox.

To start using flexbox, you need to set the display property of an element either to flex or inline-flex to make your elements inline.

The distinction here is similar to the distinction between block and inline-block; check out this Stack Overflow question for more details.


Flex-direction

This defines the main axis and, as a consequence, the direction in which your flex items are placed. This also allows you to change the order of items, which is used to alter the underlying HTML.

Available properties are:

  • row â€” The default. Arranges flex items left to right unless you are in a right-to-left environment due to writing mode.
  • row-reverse â€” Arranges items horizontally but in reversed order.
  • column â€” The same as row but vertical with items arranged top to bottom.
  • column-reverse â€” Displays items bottom to top.




Justify-content

The justify-content property defines the item alignment on the main axis. It decides what happens with any available free space and has some control over the alignment when items get wider than their container.

  • If the flex-direction is column or column-reverse, justify-content refers to the vertical axis,
  • If the flex-direction is row or row-reverse, justify-content refers to the horizontal axis.

Here are the values you can choose from:

  • flex-start â€” Default value. Items align towards the front of flex-direction.
  • flex-end â€” Places items at the end of flex-direction.
  • center â€” Items center horizontally.
  • space-between â€” Distributes items evenly within the container. The first towards the start, the last toward its end with even space between them.
  • space-around â€” Items are evenly placed with equal space around them. Note that it behaves like margin or padding where you have double the space between items as towards the edges.
  • space-evenly â€” Items reside evenly placed within the container but the spacing between each and toward the container edges is even.

Align-items

This property controls how items align across the cross axis.

  • If the flex-direction is column or column-reverse, align-items refers to the horizontal axis,
  • If the flex-direction is row or row-reverse, align-items refers to the vertical axis.

Here are the available values:

  • stretch â€” Default value that stretches items to fill the container.
  • flex-start, start, self-start â€” Aligns flex items at the start of the cross axis. start and self-start adhere to flex-direction and writing-mode respectively.
  • flex-end, end, self-end â€” The same as above but placing items at the end of the cross axis.
  • center â€” Items reside at the center of the cross axis.
  • baseline â€” Aligns flex items along their baselines.

Flex-wrap

The default behavior of items within a flex container is to arrange themselves in one row.
flex-wrap allows you to change that.

  • nowrap â€” The default value that places all items in one line.
  • wrap â€” If a single line isnâ€™t enough, with this, items will arrange themselves into multiple lines from top to bottom.
  • wrap-reverse â€” Same as wrap but with items ordered from bottom to top.

When items in a flex container get too wide, what should happen? This property lets you tell the browser whether it should allow flex elements to wrap onto a new line, or whether they should just shrink in order to fit on the same line. Fore more on this, check out MDN.



Align-Content

align-content : This property is responsible for controlling flex lines if there is extra space available on the cross axis.
You need more than one row of items for the values below to take effect.




II. Flex properties on the flex items

Order

This rule apply to flex items instead of the container. The order property controls in which order items appear inside their container.

The default value for all flex items is order: 0
If you want to move a particular item to the front or back of the line, you can do so by giving it a value like 1 or -1.
This also works across row or column boundaries unlike row-reverse or column-reverse which will reverse the order per line individually.

    <div style="order: 1">4</div>

Align-self

This property allows you to overwrite the value of align-items for particular elements. It accepts the same values as align-items.

  • flex-start: cross-start margin edge of the item is placed on the cross-start line
  • flex-end : cross-end margin edge of the item is placed on the cross-end line
  • center : item is centered in the cross-axis
  • baseline : items are aligned such as their baseline are aligned
  • stretch (default): stretch to fill the container (still respect min-width/max-width)



Other properties on the flex items

  • flex-basis - This lets you manually set the size of a flexbox elementâ€™s content box. At first glance it may seem like itâ€™s synonymous with the width property, but this isnâ€™t always the case. For more details, read this Stack Overflow question.

  • flex-grow - This lets you set proportional sizes of flex items within the same container. For example, an element with a flex-grow value of 2 will take up twice as much space as an element with a flex-grow value of 1.

  • flex-shrink - Similar to flex-grow, but for shrinking elements when thereâ€™s not enough space. A useful demo of this property is available at CSS Tricks.

  • flex - This is a shorthand which lets you set flex-grow, flex-shrink, and flex-basis all in one line (similar to the border property, which lets you set border-width, border-style, and border-color).
